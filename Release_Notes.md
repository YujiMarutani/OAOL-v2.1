## OAOL v2.1 — Autonomous Orchestration Operating Layer

OAOL (Autonomous Orchestration Operating Layer) v2.1 is the canonical release of the multi‑agent orchestration architecture designed to transform human sovereign intent into executable task graphs, manage dynamic phase states, detect instability, harmonize multi‑agent outputs, recover from failures, and retrieve canonical knowledge from immutable academic sources.

This release provides the complete JSON‑based canonical specification for OAOL v2.1.

## 🧠 Architecture Overview

OAOL v2.1 is structured into four layers:

- **L4 — Sovereign Intent Layer**  
  Human goals, constraints, ethical boundaries, priority structures.

- **L3 — Orchestration & Semantic Layer**  
  Task graph generation, phase management, agent routing.

- **L2 — Boundary & Security Layer**  
  Phase state machine, agent channels, audit hooks.

- **L1 — Execution Layer**  
  External bindings, simulations, canonical knowledge retrieval.
